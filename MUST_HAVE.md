
### 第一部分：语言特性（使用者视角，这门语言必须能跑什么代码？）

作为一门合格的现代脚本语言，哪怕是玩具级别，也必须包含以下 6 大语法功能：

#### 1. 基础数据类型 (Data Types)
动态语言的变量没有类型，但“值”是有类型的。你必须支持以下最基本的数据类型：
*   **整数 (Integer)**：例如 `10`, `-5`。（浮点数前期可省略，避免精度处理的麻烦）。
*   **布尔值 (Boolean)**：`true` 和 `false`，用于逻辑判断。
*   **空值 (Null / Nil)**：表示“没有值”或函数默认返回状态。
*   **字符串 (String)**：例如 `"hello world"`，必须支持字符串拼接 (`+`)。

#### 2. 变量绑定与环境 (Variables)
*   **变量声明**：必须能将一个值绑定到一个名字上。例如 `let x = 10;` 或 `var name = "Golang";`。
*   **变量寻址/重赋值**：声明后的变量可以被使用，也可以被重新赋值 `x = 20;`。

#### 3. 表达式与运算符 (Expressions & Operators)
这是语言“计算”的核心，必须支持按**优先级**正确计算：
*   **算术运算**：`+`, `-`, `*`, `/`（比如 `1 + 2 * 3` 必须算出 `7` 而不是 `9`）。
*   **比较运算**：`==`, `!=`, `<`, `>`（返回布尔值）。
*   **前缀运算**：`-` (取负，如 `-5`)，`!` (逻辑非，如 `!true`)。

#### 4. 控制流 (Control Flow)
没有控制流的语言只能算计算器，必须包含：
*   **条件分支 (If-Else)**：
    ```javascript
    if (x > 10) { print("large"); } else { print("small"); }
    ```
*   **循环 (Loop)**：只要支持最基础的 `while` 或者 `for` 即可，这是图灵完备的基础。

#### 5. 一等公民函数与闭包 (Functions & Closures)
这是让你的语言走向高级的**绝对核心**。必须支持：
*   **函数定义与调用**：`let add = fn(x, y) { return x + y; }; add(2, 3);`
*   **隐式/显式 Return**：函数遇到 `return` 必须立刻终止并返回值。
*   **闭包 (Closure) [极其重要！]**：内部函数必须能记住并访问外部函数的变量。
    ```javascript
    let makeAdder = fn(x) {
        return fn(y) { return x + y; }; // 内部函数必须能拿到外部的 x
    };
    ```

#### 6. 内置函数 (Built-ins)
至少需要一个 `print()` 或 `puts()` 函数,同时必须支持从标准输入读取。因为如果无法将结果读取/输出到控制台，你就无法验证语言是否运行正常。

### 第二部分：解释器底层架构（开发者视角，你需要用 Go 写哪些核心模块？）

为了实现上述的语言特性，你的 Go 代码项目中**必须包含以下 5 个核心模块（缺少任何一个都跑不起来）**：

#### 1. 词法分析器 (Lexer / Scanner)
*   **功能**：把用户输入的字符串源码（例如 `let x = 5;`），逐字读取，切分成一个个的词法单元（Token）。
*   **必须产出**：输出一个 Token 序列。例如 `[TOKEN_LET, TOKEN_IDENTIFIER("x"), TOKEN_ASSIGN, TOKEN_INT("5"), TOKEN_SEMICOLON]`。
*   **避坑**：必须跳过空格、换行符和注释。

#### 2. 语法分析器 (Parser)
*   **功能**：读取 Lexer 输出的 Token 序列，根据语法规则，将其组装成一棵**抽象语法树 (AST)**。
*   **必须包含的技术**：
    *   **递归下降解析 (Recursive Descent Parsing)**：处理语句（Statement），如 `let` 语句、`return` 语句。
    *   **普拉特解析 (Pratt Parsing / Top-Down Operator Precedence)**：专门用来处理表达式的优先级。**（不使用这个算法，你几乎无法优雅地处理 `1 + 2 * 3` 这种优先级问题）**。
*   **必须产出**：一棵树状数据结构。
    *(注：在 Golang 中，AST 节点通常被定义为实现了 `Node` 接口的 Struct。)*

#### 3. 对象系统 (Object System)
在动态语言中，由于你不知道变量是什么类型，所以你需要一个统一的“盒子”来装所有类型的值。
*   **功能**：在 Go 语言里定义一套结构体，用来表示你语言运行时的“值”。
*   **必须实现**：
    *   在 Go 里定义一个通用的 `Object` 接口。
    *   实现 `IntegerObject`、`BooleanObject`、`StringObject` 等等。
    *   每个 Object 必须能返回自己的类型标识（比如用于报错提示）。

#### 4. 环境与作用域 (Environment)
这是解释器的“记忆系统”，非常关键。
*   **功能**：本质上是一个哈希表（Go 里的 `map[string]Object`），用来存储**变量名**和对应**值（Object）**的映射关系。
*   **必须包含的结构**：环境必须是一个**链表**结构！它不仅包含当前的 `map`，还必须包含一个指向**外部环境（Outer Environment）**的指针。
*   **原因**：当代码查找变量 `x` 时，如果当前作用域找不到，必须顺着指针去外层作用域找。**这是实现闭包和块级作用域（Block Scope）的唯一正确做法。**

#### 5. 树遍历求值器 (Evaluator)
这是整个解释器的心脏。
*   **功能**：写一个类似于 `func Eval(node ast.Node, env *Environment) object.Object` 的函数。
*   **必须实现**：
    *   利用 Go 的 `switch-case` 类型断言，判断当前 AST 节点是什么类型。
    *   如果是 `+` 号节点：分别递归调用 `Eval()` 算出左边和右边的值，然后在 Go 层面把两个数字加起来，包装成新的 `IntegerObject` 返回。
    *   如果是 `If` 节点：先 `Eval()` 条件部分，如果是真，则递归遍历执行主体块的 AST。
    *   如果是 `Return` 节点：必须抛出一个特殊的标记，让外层的递归能够捕获并立刻中断执行。
    *   **错误处理机制**：在遍历过程中遇到问题（比如 `1 + "hello"`），必须生成一个 `ErrorObject`，并且立刻停止后续执行并打印错误。

### 总结：完整工作流图示

当你写完这 5 个模块后，一段代码在你的解释器里的生命周期是这样的：

源码 `"let x = 1 + 2; print(x);"`
⬇️ *(被 Lexer 拆解)*
Tokens `[LET, IDENT(x), ASSIGN, INT(1), PLUS, INT(2)...]`
⬇️ *(被 Parser 组装)*
AST (`LetStatement { Name: "x", Value: BinaryExpr { Left: 1, Op: +, Right: 2 } }`)
⬇️ *(被 Evaluator 遍历计算，存入 Environment，操作 Object)*
在控制台输出：`3`

这就是一个**动态类型 + 树遍历解释器**的完全体形态。只要你老老实实按照这 5 个模块，一步一步用 Go 把对应的 struct 和 func 写出来，一门拥有现代特性的编程语言就从你手中诞生了！
